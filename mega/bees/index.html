Bee Lab

The same pixel world, one scale down. A bee is ~3px — so we bake the look (a tiny seed-deterministic micro-atlas, portable like a sprite genome) and simulate the motion (boids-lite + curl-noise, not a baked loop). Move your cursor over the field to lead the swarm: it reacts, and never repeats.

Swarm

Forces

cursor / touch = the flower · the swarm follows but won't collapse onto it

Why this and not a looped animation

A baked loop either syncs every bee (uncanny) or rides fixed rails that ignore you. This is N independent agents over a shared turbulence field — it reacts to the cursor and never repeats.

The split that keeps it cheap

Appearance is baked once. beeAtlas(seed) → headings × 2 wingbeat frames of {x,y,c} pixels. Per frame we only pick a heading bin from velocity and stamp — never recompute pixels. Motion is a struct-of-arrays fixed-timestep sim (Swarm.step), the exact shape a WebGPU compute pass / Rust loop wants. Hundreds of bees here in plain JS; the GPU port carries tens of thousands (see fluoddity, 40–120k agents at 60fps).

HTTP API — the portable bake

GET /bees/api/atlas.json?seed=&headings= → the full micro-atlas an engine uploads to a texture.
GET /bees/api/bee.svg?seed=&heading=&wing=&scale= → one bee as an image.
