# case — mysteries read off the civic web

Live at **tide.mino.mobi/case**. Goss's sister site: procedurally generated murder mysteries
over the same civic web, with a solvability oracle and a **chain** — every closed case scars
the web the next case grows from. Pure static, no build, zero deps beyond the goss kernel.

```bash
node tide/case/test/casegen.selftest.mjs   # 23 checks, 60 cases certified across 10 substrates
```

## The thesis

Classic Clue is famously **motiveless** — person + room + weapon, no *why*. The goss web
inverts that: motive is the native resource. Every drama already carries its evidence, so a
case is generated by picking a victim at the intersection of several dramas, promoting ONE
grievance to the actual cause, and letting the rest stand as **red herrings with genuine
motive but innocence** — the expensive thing in hand-authored mysteries, free here.

## How a case is built (`casegen.js`)

| layer | where it comes from |
|---|---|
| **MOTIVE** | `grievancesAgainst`: dramas involving the victim (AFFAIR · TRIANGLE · MATCH · …), rivals (same job, near-identical webs), wronged partners (sparks vs the couple map), household succession |
| **TIME** | a deterministic day of six watches; each soul's hats laid out as a schedule (home · work · thirds), seeded off **stable identity** — a survivor keeps the same routine across every case of a chain |
| **OPPORTUNITY** | the crime happens at the victim's scheduled place on some watch; a suspect's alibi is their own scheduled place |
| **WITNESSES** | only **independent** witnesses clear a name (ties weaker than 1.5 — Granovetter's weak ties as evidentiary value). The household swearing to you is worth exactly what kin's word is worth. The culprit is never corroborated. |
| **MEANS** | items typed by place role (clinic sedative, chopshop spanner, canteen ration…); access is a hat query. The planted item is drawn from the culprit's access set among those that keep ≥2 suspects on the board (pacing is a generation constraint); a culprit with no means-role hats falls back to the cord that narrows nothing. |
| **CLUES** | body → rumor → the coroner's finding → the canvass (one alibi per suspect) → sightings (descriptors: age band, ward colours, trade) → traces (a member's token, a household thread) → grounded last resorts (a passerby on the adjoining watch; the reluctant eyewitness whose schedule really brushed the scene) |
| **THE ORACLE** | the planted clue list is replayed cold as a deductive closure (eliminations only) and must converge on **exactly the culprit**, never eliminating them; the generator retries other watches before giving up. Every shipped case carries `solvable: true` because unsolvable ones never leave the kernel. |

## The chain — a seed is a casebook

Closing a case removes victim + culprit; the relational layers (ties → tribes → romance →
tension → dramas) re-run over the **survivors** while demographics stay **cast-once** —
nobody is renamed by a death (`buildView` carries `orig` identity through the re-index; the
selftest pins name stability across a five-case chain). Case n+1 is generated from the
scarred web: grievances shift, tribes re-knit, vitality ticks as souls go. Same seed ⇒ same
casebook, forever — so `?sub=&seed=&case=n` is a permalink (prior cases replay closed).

Every baked nave seed and the econ towns sustain at least a 6-case chain (pinned).

## The viewer (`index.html` + `app.js`)

The magistrate's file: victim dossier, the clue feed revealed lead by lead, the suspect
board with **live eliminations computed only from revealed clues**, accuse (three dismissed
accusations and the case goes cold — the truth outs either way, the chain continues), the
morgue, and an **ego web** canvas — victim centred, suspects ringed in tribe colours with
faction rings, ties at true weight, claim places on dotted spokes, the scene pinned. The
nave substrate always uses the **unified floor** (the casebook's canon — see
`../goss/UNIFIED.md`: sealed wards are too thin for whodunits; clump-souls have no
independent witnesses, so alibis can neither be trusted nor falsified).

## Files

| file | role |
|---|---|
| `casegen.js` | the kernel — schedules, grievances, ground truth, clue planting, the solvability oracle, the chain. Pure, zero-DOM. |
| `index.html` + `app.js` | the viewer — presentation only; all logic is kernel output |
| `docs/index.html` | **live documentation** (`/case/docs/`) — the whole goss→case stack explained, with figures computed in-browser by the real kernels (society vitals, sealed-vs-unified bars, case № 1's closure replayed clue by clue, a four-case chain run). Change seed/substrate in its topbar and the page recomputes. |
| `test/casegen.selftest.mjs` | 23 checks — determinism, closure ≡ {culprit}, culprit never eliminated, alibi honesty, pacing, chain scars, cast-once identity, 10-substrate coverage |

## Where this goes next

- **The escaped killer**: a cold case could leave the culprit free on the web — eligible to
  kill again — at the cost of permalink determinism (chain state would depend on play).
  Deliberately not done; the deterministic casebook is worth more for now.
- **hoopy retelling**: cases are structured JSON with prose sketch lines — exactly the
  borges/v096 shape (procedural bones, model retelling, frozen per seed).
- **Route-aware opportunity**: schedules currently teleport between watches; pricing routes
  (hoop's nav) would make "could they have crossed the floor in time?" a real clue axis.
- **The other genres**: maintenance mysteries over the forge flow web (a planted fault,
  symptoms propagating through conserved books — whodunit for machines) and romance arcs
  (the tie graph run forward) are the same machine on different graphs.
