who blocks them ⛒

everyone blocking a user

Give one handle Y and see everyone who blocks Y — the full count and the list, newest blocks first. Unlike the other gc tools (which read raw block records for a bounded set of accounts), a network-wide blocker list can't come from raw records — blocks live in each blocker's own repo, scattered across all of Bluesky. This reads the firehose-built block index at clearsky.services through our API.

Index-backed. Counts and membership are as fresh and complete as Clearsky's index, which mirrors the firehose. Direct blocks only (a subscribed moderation block-list isn't a per-account block). This is the same data the /api/gc/blockers endpoint serves — agents can call it directly.
