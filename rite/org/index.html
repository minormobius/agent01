The API

Everything on this page is also a public JSON API (CORS open). The engine is deterministic: the same seed + node id returns the same company forever, so a URL is a permanent address inside an unbounded org.

curl 'https://rite.mino.mobi/api/org?seed=acme&vertical=corp&shape=pyramid&depth=4'   # tree + people + performance
curl 'https://rite.mino.mobi/api/org?seed=acme&vertical=corp&shape=flat&depth=4'      # same seed, worse shape
curl 'https://rite.mino.mobi/api/org/node?seed=acme&vertical=corp&shape=fractal&id=r.2.1.0'  # the infinite lens
curl 'https://rite.mino.mobi/api/org/person?seed=acme&vertical=corp&id=r.2.1'   # one person + metrics
curl 'https://rite.mino.mobi/api/org/verticals'   # catalog of verticals, shapes, attributes
verticalthe industry — its rank ladder and title vocabulary. corp, startup, military, feudal, crime, monastic, academic, ecclesiastic.
shapethe topology laid over it: pyramid, tall, flat, wide, matrix (dotted lines), cellular, fractal (endless).
seedany string. Omit it and the server mints one (returned with a permalink).
depth1–12, default 4. The bounded tree stops here; /node keeps going past it.
namesoptional — override the culture the people are named from (any names culture, e.g. norse, nihon, desertic).
id/node only: a node address like r.2.1.0 (root, its 3rd report, that one's 2nd, that one's 1st). Returns the node + one level of reports.

Each org is generated under a per-seed charter — which of the vertical's departments this particular company actually runs, plus its own name. The vertical supplies the titles (a corp’s Senior Director of Platform, a legion’s Colonel, 3rd Regiment, a duchy’s Margrave of Aldermoor); the shape supplies the tree (a flat org drops middle management, a matrix adds dotted-line bosses, a cellular one compartments into small crews). The people are minted by the names engine, so a crime family reads Salvatore the Rusted and a duchy reads Roderick the Grim.

People & performance. Every box holds a person — deterministic from the seed: demographics, a work-triad (craft / drive / wit — the hands, the fire, the mind) expressed into nine attributes, a temperament (Architect, Operator, Grinder…), a quirk or two, and a vocation drawn from the same civic verbs the wider world uses. Then the whole tree is rolled up into how the org performs: a boss's leadership multiplies their reports, an overloaded span leaks throughput, every management layer skims an overhead tax, and morale flows down from manager quality and workload. So the same people under a different shape score differently — a flat org saves the depth tax but overloads its managers into a Failing grade; a tall one keeps spans sane. Tap any box for its person; use the colour selector to paint the whole chart by morale, output, competence, manager load, or flight risk. The org grade borrows econ's vitality tiers (Thriving · Healthy · Stable · Fragile · Failing).

The infinite chart. A normal org bottoms out at the individual contributor. This one wraps: ask /node for any leaf and the clerk turns out to be the apex of their own shadow sub-organisation, with its own C-suite, its own middle managers, its own clerks — all the way down. Click drill on any node below to fall through it.

Runs entirely client-side on this page (the browser imports the same engine.js the worker serves; the people are person.js, the diagram is diagram.js, canvas, no deps) — the API exists for everything that isn’t a browser. Part of rite.mino.mobi.
