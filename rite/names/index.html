names.

One seed → one coherent set of 300 names: thematically linked, all unique, all distinct. A rite surface · sister to the name-family essay · API

The API

Everything on this page is also a public JSON API (CORS open). The engine is deterministic: the same seed returns the same set forever, so a URL is a namebook.

curl 'https://rite.mino.mobi/api/names?seed=tabard&culture=norse&setting=classical&kind=given&count=300'
curl 'https://rite.mino.mobi/api/names?seed=frontier&culture=steppe+veil&setting=scifi&kind=full&count=120'
curl 'https://rite.mino.mobi/api/names/cultures'   # catalog of cultures / settings / kinds
seedany string. Omit it and the server mints one (returned in the response with a permalink).
culturea phonotactic wardrobe. Blend two with +, e.g. norse+romance.
settingregister transform: classical, fantasy, scifi, fey, wasteland.
kindgiven, family, place, or full.
count1–1000, default 300.

Every set is generated under a per-seed charter — a sub-dialect of the culture (favorite sounds Zipf-boosted so they recur, a fixed handful of ending morphemes, register shifts adopted set-wide). That's what makes 300 names read as one world rather than 300 rolls of the same dice. Uniqueness is enforced hard: no duplicates, no name within edit distance 1 of another, no name a prefix of another.

Epithets (kind=full) are generated too, not listed: each setting carries a small grammar (the {times}-{part} → the Twice-Buried, {count}-{body} → Nine-Fingers, {noun}{bond} → Stormsworn…) whose word banks and templates the charter subsamples per seed — and the of {place} template mints a toponym from the same charter, so a hero can hail from a place that sounds like their own world. No epithet repeats within a set.

Runs entirely client-side on this page (the browser imports the same engine.js the worker serves) — the API exists for everything that isn't a browser. Part of rite.mino.mobi.
